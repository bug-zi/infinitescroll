# 无限画卷功能完成度

更新时间：2026-05-07

## 已完成

- 三栏工作台 UI：左侧画卷列表、中间画卷工作台、右侧图片详情。
- Supabase 数据库连接：读取/写入 `scrolls`、`scroll_images`、`generation_jobs`、`generation_logs`。
- Supabase Storage：生成图片上传到 `scroll-images` bucket，数据库保存公网 URL。
- DeepSeek 主题优化：创建画卷时生成长期画卷提示词。
- 真实图片生成：首图普通生成，后续优先使用 Image Edit 扩图续画。
- 衔接策略：后续图保存完整图、重叠区、可见区，预览只展示非重复可见区。
- 像素级重叠校准：上一张右侧重叠区会覆盖到下一张左侧重叠区。
- 本地自动生成调度器：`npm run dev:api` 启动后每 30 秒扫描到期画卷。
- 手动立即生成：当前画卷可主动触发下一张。
- 任务队列：生成任务写入 `generation_jobs`，记录 `running / succeeded / failed`。
- 控制台页面：显示画卷、图片、今日生成、运行任务、失败任务、日志和衔接风险。
- 图片操作：删除、重新生成、前插、后插。
- 中间图片重生成：从被重生成图片开始自动逐张重绘到尾部。
- 插入图片：从插入位置开始自动逐张重绘到新的尾部，避免后续段沿用旧衔接。
- 批量重绘保护：如果任意一张生成失败，会暂停画卷并写入错误日志，避免继续生成错位画面。
- 并行多个画卷：调度器按 `MAX_CONCURRENT_JOBS` 扫描并并行处理多个到期画卷，同一画卷仍用 running job 锁避免并发写入。
- Vercel Cron API：`api/cron/generate.ts` 已同步扩图、像素重叠、并行处理、running job 锁、超时和失败释放逻辑。
- 观赏模式：点击画卷段进入可缩放、拖拽、键盘移动的长卷查看。
- 失败任务重试：控制台失败任务可以一键重试。
- 画卷编辑：标题、主题、DeepSeek 优化提示词可以保存。
- 衔接质量评分：新生成的衔接图会保存评分并在低分时标记风险。
- 部署环境检查：`npm run check:deploy-env` 可检查 Vercel 所需环境变量。
